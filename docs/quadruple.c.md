### 三元表达式 `(a1 && a1[0]) ? a1 : "_"` 详细解释

```
(a1 && a1[0]) ? a1 : "_"
```

| 子表达式 | 含义 |
|----------|------|
| `a1` | 检查指针 `a1` 是否为 `NULL`（空指针）。C 语言中 `NULL` 为 0（false），非空指针为 true。 |
| `a1[0]` | 检查字符串 `a1` 的第一个字符是否为 `'\0'`（字符串结束符）。空字符串 `""` 的 `a1[0]` 是 `'\0'`（即 0 / false）。 |
| `&&` | 逻辑与，短路求值。如果 `a1 == NULL`，则不再执行 `a1[0]`，直接返回 false，从而避免对空指针解引用导致的段错误（segfault）。 |
| 整体条件 | 当 `a1` 既不是 `NULL`，也不是空字符串 `""` 时，条件为真。 |
| `? a1` | 条件为真时，使用原字符串作为字段值。 |
| `: "_"` | 条件为假时，用下划线 `"_"` 作为占位符，表示"此槽位未使用"。 |

### quad_backpatch() 的应用场景

回填（backpatching）技术用于处理**前向跳转**目标在生成跳转指令时尚未已知的情况：

1. **条件分支（if）**：当解析到 `if` 语句的条件时，生成了 `jmp _ _ _`（跳转到 else 部分或结束），但此时还不知道 else 或结束位置的确切标号。于是先 `quad_emit` 生成占位四元式，记录其序号，等到解析完 then 部分、确定了跳转目标标号后，再调用 `quad_backpatch(c, saved_idx, label)` 把标号回填进去。

2. **循环（while）**：当解析到 `jmp _ _ _` 需要跳回循环开头时，回跳的目标（L_loop 标号）可能已经通过 `OP_LABEL` 产生。但也有一些场景需要回填。

3. **一般流程**：
   ```
   int jmp_idx = quad_emit(c, OP_JMP, NULL, NULL, "_");  // 生成占位跳转，result 暂时为 "_"
   // ... 解析后续代码，确定目标标号 ...
   quad_backpatch(c, jmp_idx, "L_end");  // 将占位的 "_" 替换为 "L_end"
   ```

4. **边界保护**：第 49 行的 `if (quad_idx >= 0 && quad_idx < c->quad_count)` 是防御性编程：当 `quad_emit` 因容量满而返回 -1 时，`quad_backpatch` 不会尝试对 `quads[-1]` 写入，避免数组越界和段错误。
