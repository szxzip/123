| 5 | `static int is_int_const(const char *s) {` | 函数定义开始。`static` 限定作用域为本编译单元，防止外部链接冲突。参数 `s` 为待检查的 C 字符串指针。返回 1 表示是整型常量，0 表示不是。 |

| 7 | `    if (s[0] == '-') s++;` | 处理负号：如果字符串以 `-` 开头，将指针向前移动一位跳过负号，后续仅检查剩余部分是否为纯数字。例如 `"-10"` 跳过负号后检查 `"10"`。 |
| 8 | `    if (s[0] == '\0') return 0;` | 再次检查：跳过负号后如果直接到达字符串结尾（即原字符串为 `"-"` 或 `"-"` 后无内容），说明不是合法常数，返回 0。 |
| 9 | `    for (; *s; s++)` | 遍历字符串的剩余每个字符。`*s` 在 C 中为真值只要它不是 `'\0'`，循环将持续到字符串结尾。每次迭代指针后移一位。 |
| 10 | `        if (*s < '0' \|\| *s > '9') return 0;` | 字符范围检查：当前字符的 ASCII 值是否在 `'0'`（48）到 `'9'`（57）之间。任何不在该范围的字符（字母、小数点、符号等）都会导致返回 0。这意味着该函数只接受纯整数，不支持浮点数。 |
| 11 | `    return 1;` | 所有字符均通过数字检查，该字符串是合法的整型常量字符串（形如 `"123"` 或 `"-456"`），返回 1。 |
| 12 | `}` | 函数结束。 |
| 13 | （空行） | 排版分隔。 |
| 14 | `// 解析整常数值` | 注释：简洁说明 `const_val()` 为 `atoi` 的封装，将整常数字符串转为机器整数。 |
| 15 | `static int const_val(const char *s) {` | 函数定义。同样使用 `static` 限定作用域。返回解析出的 `int` 值。 |
| 16 | `    return atoi(s);` | 直接调用 C 标准库 `atoi`（ASCII to Integer）。由于调用前已经通过 `is_int_const()` 验证，这里不会出现解析失败的情况。`atoi` 对 `"-10"` 返回 `-10`，对 `"42"` 返回 `42`。 |
| 17 | `}` | 函数结束。 |
| 18 | （空行） | 排版分隔。 |

## 1. 常量折叠 (`fold_constants`)

| 行号 | 代码 | 讲解 |
|------|------|------|
| 19 | `// ========== 1. 常量折叠 ==========` | 注释分隔符，标记常量折叠代码段的开始。 |
| 20 | `// * 扫描四元式序列，对纯常数运算直接计算结果` | 注释：概述常量折叠的核心思想——遍历所有四元式，如果某条运算的两个操作数在编译期已知（都是常量字符串），则直接计算其结果，替换为赋值语句。 |
| 21 | `// *   (* 2 5 t1)  →  (:= 10 _ t1)` | 注释：示例——乘法运算 `2 * 5` 在编译期算出 10，四元式 `(* 2 5 t1)` 被替换为 `(:= 10 _ t1)`。`_` 表示 arg2 不再使用。 |
| 22 | `// *   (+ 3 4 t2)  →  (:= 7 _ t2)` | 注释：加法示例。 |
| 23 | `// *   (- 10 3 t3) →  (:= 7 _ t3)` | 注释：减法示例。 |
| 24 | `// *   (/ 8 2 t4)  →  (:= 4 _ t4)` | 注释：除法示例。注意如果除数为 0 则不会折叠，保持原样（见第 40 行）。 |
| 25 | `static int fold_constants(Compiler *c) {` | 函数定义开始。`static` 限定作用域。参数为编译器上下文指针。返回值：`1` 表示本轮至少折叠了一条四元式（有变化），`0` 表示没有可折叠的。该返回值用于 `optimize_run` 判断是否需要继续迭代。 |
| 26 | `    int changed = 0;` | 初始化变更标志为 0。在整个遍历过程中，一旦成功折叠任意一条四元式，该标志就置为 1。 |
| 27 | `    for (int i = 0; i < c->quad_count; i++) {` | 开始遍历编译器中的所有四元式。`c->quad_count` 是当前四元式的数量。循环变量 `i` 从 0 到 `quad_count-1`。 |
| 28 | `        Quadruple *q = &c->quads[i];` | 获取当前四元式的指针。`c->quads` 是四元式数组（在 `grammar.h` 中定义），用下标 `i` 取地址得到指向该四元式的指针，方便后续读写。 |
| 29 | `        if (q->op == OP_ADD \|\| q->op == OP_SUB \|\|` | 条件判断第一行：当前四元式是否为算术运算——加法 (`OP_ADD`)、减法 (`OP_SUB`)、乘法 (`OP_MUL`)、除法 (`OP_DIV`)。只有这些二元运算才可能进行常量折叠。 |
| 30 | `            q->op == OP_MUL \|\| q->op == OP_DIV) {` | 条件判断的续行（C 语言允许跨行）。合起来判断 `op` 是否属于四则运算。比较操作 (`OP_JE`、`OP_JL` 等) 和逻辑操作 (`OP_AND_OP`、`OP_OR_OP` 等) 不在此列。 |
| 31 | `            if (is_int_const(q->arg1) && is_int_const(q->arg2)) {` | 关键过滤：只有当 `arg1` **和** `arg2` 都是整数常量时，才能折叠。如果任一操作数是变量名（需要运行时才能确定值），则跳过该条四元式。 |
| 32 | `                int a = const_val(q->arg1);` | 将 `arg1` 字符串解析为 `int` 值并存入变量 `a`。例如 `"5"` → `5`、`"-3"` → `-3`。 |
| 33 | `                int b = const_val(q->arg2);` | 将 `arg2` 字符串解析为 `int` 值并存入变量 `b`。至此编译期获得了两个操作数的具体数值。 |
| 34 | `                int r = 0;` | 声明结果变量 `r` 并初始化为 0。后续 `switch` 语句的各个分支将在此变量中存入对应运算的计算结果。 |
| 35 | `                switch (q->op) {` | 根据操作码分派到不同的运算分支。`switch` 比多个 `if-else` 更清晰高效。 |
| 36 | `                    case OP_ADD: r = a + b; break;` | 加法分支：计算 `a + b`，结果存入 `r`。`break` 跳出 switch 继续执行替换逻辑。 |
| 37 | `                    case OP_SUB: r = a - b; break;` | 减法分支：计算 `a - b`。注意与表达式树不同，四元式中 arg1 是被减数、arg2 是减数。 |
| 38 | `                    case OP_MUL: r = a * b; break;` | 乘法分支：计算 `a * b`。 |
| 39 | `                    case OP_DIV:` | 除法分支的开始标签。没有直接写计算语句，因为需要先进行除零检查。 |
| 40 | `                        if (b == 0) continue;` | **除零安全检查**：如果除数 `b` 为 0，则退出本次循环（`continue` 跳到第 27 行的 `for` 下一轮迭代，跳过第 41-48 行的替换操作）。该四元式被保留原样（运行时将会报错或产生未定义行为，由运行时环境处理），但不做编译期折叠。 |
| 41 | `                        r = a / b;` | 除数非零时执行整数除法。注意 C 语言的整数除法会截断小数部分（向零舍入），这与语言语义一致。 |
| 42 | `                        break;` | 跳出 `switch` 语句。 |
| 43 | `                }` | `switch` 语句结束。此时 `r` 中保存了运算结果。 |
| 44 | `                // 替换为赋值四元式` | 注释：说明下面的代码是将原算术运算四元式改写为赋值四元式。 |
| 45 | `                q->op = OP_ASSIGN;` | 将四元式的操作码从算术运算（`OP_ADD`/`OP_SUB`/`OP_MUL`/`OP_DIV`）修改为赋值操作 `OP_ASSIGN`。这是"折叠"的核心——不再需要运行时计算，只需直接赋值。 |
| 46 | `                snprintf(q->arg1, MAX_STR, "%d", r);` | 将计算结果 `r`（整数）格式化写入 `arg1` 字段。`MAX_STR` 是字符串缓冲区的最大长度（定义在 `grammar.h` 中），`snprintf` 保证不会溢出。例如 `r=7` 则 `arg1` 变为 `"7"`。 |
| 47 | `                q->arg2[0] = '_'; q->arg2[1] = '\0';` | 将 `arg2` 设置为占位符 `"_"`（空字符串）。手动设置两个字符：`'_'` 和结尾的 `'\0'`。因为赋值操作 `OP_ASSIGN` 的语义是 `(:= arg1 _ result)`，arg2 不使用。 |
| 48 | `                changed = 1;` | 设置变更标志为 1，告知调用方本轮发生了优化（在 `optimize_run` 中用于决定是否继续下一轮迭代）。 |
| 49 | `            }` | `is_int_const` 条件块结束。 |
| 50 | `        }` | `op == OP_ADD/OP_SUB/OP_MUL/OP_DIV` 条件块结束。 |
| 51 | `    }` | `for` 循环结束——全部四元式扫描完毕。 |
| 52 | `    return changed;` | 返回变更标志。如果整个遍历过程中至少折叠了一条四元式则返回 1，否则返回 0。 |
| 53 | `}` | 函数结束。 |

## 2. 常量传播 (`propagate_constants`)

| 行号 | 代码 | 讲解 |
|------|------|------|
| 55 | `// ========== 2. 常量传播 ==========` | 注释分隔符，标记常量传播代码段的开始。 |
| 56 | `// * x := 常数 ;  ... ; y := x op z   →  y := 常数 op z` | 注释：常量传播的第一个模式——当 arg1 是一个已知常量值的变量时，替换为常量。例如 `t := 5` 然后 `y := t + z` 变为 `y := 5 + z`，后续常量折叠可进一步优化。 |
| 57 | `// * x := 常数 ;  ... ; y := z op x   →  y := z op 常数` | 注释：常量传播的第二个模式——当 arg2 是一个已知常量值的变量时，同样替换。两个位置（arg1 和 arg2）都需检查。 |
| 58 | `static int propagate_constants(Compiler *c) {` | 函数定义开始。返回值含义与 `fold_constants` 相同：1 表示发生了传播（有变化），0 表示没有。 |
| 59 | `    int changed = 0;` | 初始化变更标志为 0。 |
| 60 | `    int i, j;` | 声明两个循环变量。`i` 用于外层遍历四元式寻找常量赋值语句；`j` 用于内层从赋值点向后扫描使用点。C89 风格声明（在函数体开头），与 `changed` 分开便于阅读。 |
| 61 | `    for (i = 0; i < c->quad_count; i++) {` | 外层循环：遍历所有四元式，寻找形如 `(:= 常数 _ 变量)` 的赋值语句作为传播源。 |
| 62 | `        Quadruple *q = &c->quads[i];` | 获取当前四元式 `q` 的指针。 |
| 63 | `        // 找到常数赋值: (:= C _ var)` | 注释：描述要匹配的模式——操作码为 `OP_ASSIGN`、arg1 是常量字符串、arg2 为占位符、result 是变量名（不是常量）。 |
| 64 | `        if (q->op == OP_ASSIGN && is_int_const(q->arg1) &&` | 条件判断第一行：操作码是 `OP_ASSIGN` 且 arg1 是整数常量。arg1 是常量和 arg2 是 `_`，这两个条件同时满足才构成"常量赋值语句"。 |
| 65 | `            q->arg2[0] == '_' && !is_int_const(q->result)) {` | 条件判断续行：arg2 的首字符是 `_`（即占位符）且 result 不是常量。`!is_int_const(q->result)` 确保赋值目标是变量而不是另一个字面量（虽然理论上不应该出现 `(:= 1 _ 2)` 这种四元式）。 |
| 66 | `            const char *var = q->result;` | 保存被赋值变量的名字。这是一个指向 `q->result` 字段（在 `Quadruple` 结构体内的固定大小字符数组）的指针。后续内层循环用此变量名在其它四元式中进行匹配。 |
| 67 | `            const char *val = q->arg1;` | 保存常数值的字符串表示。例如原始四元式是 `(:= 5 _ t1)`，则 `var="t1"`, `val="5"`。内层循环将用 `val` 替换 `var` 的出现。 |
| 68 | `            // 向后搜索使用该变量的运算` | 注释：从当前赋值语句的下一条开始，向前搜索所有引用该变量的四元式。 |
| 69 | `            for (j = i + 1; j < c->quad_count; j++) {` | 内层循环开始：`j` 从 `i+1`（当前赋值语句之后的第一个四元式）开始向后遍历。这是关键设计——常量传播只向前（代码执行方向上）搜索，因为只有后续的代码才会受益于本次赋值。 |
| 70 | `                Quadruple *u = &c->quads[j];` | 获取搜索位置的四元式 `u` 的指针。用不同变量名 `u`（use site）与 `q`（definition site）区分，增强代码可读性。 |
| 71 | `                // 如果变量被重新赋值, 停止传播` | 注释：传播的终止条件之一——遇到对同一变量的重新赋值。因为新赋值会覆盖旧值，之后的代码看到的是新值而非原常量，继续传播会产生错误。 |
| 72 | `                if (u->op == OP_ASSIGN && strcmp(u->result, var) == 0)` | 条件判断：当前四元式是赋值操作且赋值目标 `result` 正是我们跟踪的变量 `var`。`strcmp` 返回 0 表示字符串相等。注意此处比较的是 `result`（赋值目标）而非 `arg1`（源操作数），因为 `(:= something _ var)` 是对 `var` 的重新定义。 |
| 73 | `                    break;` | 立即跳出内层 `for` 循环，停止对该变量的传播搜索。该 `break` 只影响内层循环，外层循环继续处理下一条赋值语句。 |
| 74 | `                // 如果变量被用作操作数, 替换为常数` | 注释：终止条件之一之外，检查变量是否作为算术运算的操作数被使用（这是传播的目标位置）。 |
| 75 | `                if (u->op == OP_ADD \|\| u->op == OP_SUB \|\|` | 条件第一行：仅对四则运算四元式进行替换。与常量折叠相同，只有 `OP_ADD`、`OP_SUB`、`OP_MUL`、`OP_DIV` 会使用算术操作数。 |
| 76 | `                    u->op == OP_MUL \|\| u->op == OP_DIV) {` | 条件续行：完成四种运算的判断。如果是这四种运算之一，继续检查操作数是否引用了被传播的变量。 |
| 77 | `                    if (strcmp(u->arg1, var) == 0) {` | 检查 arg1（第一个操作数）是否引用了目标变量。`strcmp` 比较 `u->arg1` 与 `var`，如果相等（返回 0），说明该运算的左操作数就是之前被赋常量的那个变量。 |
| 78 | `                        snprintf(u->arg1, MAX_STR, "%s", val);` | 执行替换：将 `arg1` 字段的内容覆写为常量字符串 `val`。例如 `t1` 被替换为 `"5"`。使用安全格式化函数 `snprintf` 避免缓冲区溢出。 |
| 79 | `                        changed = 1;` | 设置变更标志为 1，表示本轮发生了至少一次常量传播替换。 |
| 80 | `                    }` | arg1 替换块结束。 |
| 81 | `                    if (strcmp(u->arg2, var) == 0) {` | 检查 arg2（第二个操作数）是否引用了目标变量。例如四元式 `(+ x t1 t2)` 中 arg2 为 `t1`，如果 `t1` 已知是常量则进行替换。 |
| 82 | `                        snprintf(u->arg2, MAX_STR, "%s", val);` | 执行操作数 2 的替换，逻辑与第 78 行完全相同。 |
| 83 | `                        changed = 1;` | 设置变更标志。arg1 和 arg2 分别独立检查，各自都可能触发替换（同一四元式中两个操作数可能都是同一变量，不过概率较低）。 |
| 84 | `                    }` | arg2 替换块结束。 |
| 85 | `                }` | 运算四元式条件块结束。 |
| 86 | `                // write 使用也算` | 注释：遇到 `OP_WRITE`（输出语句）也应停止传播。因为 `write` 通常将变量的值输出，如果变量在 write 之前被重新赋值则不应停止；但这里选择保守策略——遇到 write 就停止搜索，避免跨 write 优化带来的语义争议。 |
| 87 | `                if (u->op == OP_WRITE) break;` | 如果当前四元式是写操作（`OP_WRITE`），停止向后搜索传播。`break` 跳出内层循环。 |
| 88 | `                if (u->op == OP_END) break;` | 如果遇到 `OP_END`（程序结束标记），停止传播搜索。因为超过程序结束点后的代码没有意义，也防止访问越界。 |
| 89 | `                if (u->op == OP_PROGRAM) continue;` | 如果遇到 `OP_PROGRAM`（程序起始标记，通常是第一条四元式），跳过继续搜索。`continue` 跳过本次循环体剩余部分，进入下一轮内层迭代。实际上 `j` 从 `i+1` 开始，不太可能再遇到 `OP_PROGRAM`，但保留此检查增加健壮性。 |
| 90 | `            }` | 内层 `for` 循环结束。当前变量的常量传播搜索完成（可能是正常结束，也可能是被 `break` 提前终止）。 |
| 91 | `        }` | 外层 `if`（常量赋值匹配）条件块结束。 |
| 92 | `    }` | 外层 `for` 循环结束——所有四元式扫描完毕。 |
| 93 | `    return changed;` | 返回变更标志。 |
| 94 | `}` | 函数结束。 |

## 3. 死代码消除 (`eliminate_dead_code`)

### 3.1 使用计数阶段

| 行号 | 代码 | 讲解 |
|------|------|------|
| 96 | `// ========== 3. 死代码消除 ==========` | 注释分隔符，标记死代码消除代码段的开始。 |
| 97 | `// * 删除无用的赋值: 赋值结果从未被使用 → 删掉` | 注释：死代码消除的核心逻辑——如果一条赋值语句的结果变量在后续代码中从未被读取（既不是运算的操作数，也不是 write 的参数，也不是跳转条件），则该赋值是"死"的，可以安全删除。 |
| 98 | `// * 标记每个临时变量的使用次数, 删除使用次数为 0 的赋值` | 注释：实现方法——先统计所有变量的被引用次数（use_count），然后找出 use_count 为 0 的临时变量的赋值并删除。 |
| 99 | `static int eliminate_dead_code(Compiler *c) {` | 函数定义开始。返回值：1 表示至少删除了 1 条死代码，0 表示没有可删除的。 |

### 4.1 单轮优化流程

| 行号 | 代码 | 讲解 |
|------|------|------|
| 170 | `        int changed = 0;` | 初始化本轮变更标志为 0。该标志通过按位或（`\|=`）累积三个优化阶段各自的变更状态。 |
| 171 | `        changed \|= propagate_constants(c);` | **阶段 1：常量传播**。先执行常量传播是因为它可以将变量引���替换为常量字面量，为后续的常量折叠创造机会。例如 `t:=5; y:=t+2` 传播后变为 `t:=5; y:=5+2`，接下来折叠可将后者变成 `y:=7`。如果先折叠，`y:=t+2` 中的 `t` 是变量而非常量，无法折叠。 |
| 172 | `        changed \|= fold_constants(c);` | **阶段 2：常量折叠**。在传播完成之后，对四元式中的纯常量运算进行编译期计算。此时经过传播，原本包含变量的运算可能已经全部变成常量运算（例如 `y:=5+2` → `y:=7`）。 |
| 173 | `        changed \|= eliminate_dead_code(c);` | **阶段 3：死代码消除**。在折叠完成之后执行，因为常量折叠可能让某些临时变量的赋值变得多余——例如折叠后 `t1:=5` 的常量值被直接内联到使用点，`t1` 的使用计数降为 0，该赋值成为死代码。此处在 1-2 阶段之后清理这些不再需要的赋值。 |
| 174 | `        if (!changed) break;` | **收敛检测**：如果本轮三个阶段均未产生任何变更（`changed` 仍为 0），说明优化已经收敛到不动点，提前跳出循环。这是典型的迭代优化的终止条件——当不再有任何优化可做时停止，避免无效遍历。 |
| 175 | `    }` | 迭代循环结束。 |
| 176 | `    printf("优化完成 (%d 遍)\n", pass + 1);` | 打印优化完成信息，向用户报告实际执行了多少遍迭代。`pass + 1` 将 0 基索引转为人类友好的 1 基计数。有助于调试和理解优化效果（例如输出 `优化完成 (3 遍)` 表示实际执行了 3 轮才收敛）。 |
| 177 | `}` | 函数结束，同时也是整个文件结束。 |

## 总结：优化流水线

```
optimize_run()  ── 最多 5 遍迭代 ──┐
                                    │
每遍依次执行:                        │
  propagate_constants()  → 新常量出现 │
  fold_constants()       → 计算化简  │── 任一有变更则继续下一遍
  eliminate_dead_code()  → 删除无用  │
                                    │
                              全部无变更 → 收敛退出
```

### 设计要点

1. **阶段顺序不可颠倒**：传播必须在折叠之前（传播将变量替换为常量字面量，为折叠创造条件）；折叠应在消除之前（折叠减少临时变量的引用，使死代码可被识别）。

2. **迭代的必要性**：一次传播可能暴露新的折叠机会（常量替换后出现常量+常量运算）；一次折叠可能暴露新的死代码（折叠后的临时变量不再被引用）。多遍迭代利用这种链式反应达到最大优化效果。

3. **安全性措施**：
   - 除数为 0 时不折叠（第 40 行 `continue`）
   - 常量传播在变量被重新赋值时停止（第 72-73 行 `break`）
   - 只删除临时变量，保护用户变量（第 143 行 `KIND_TEMP` 判断）
   - 遇到 `OP_WRITE` 和 `OP_END` 时停止传播（第 87-88 行）

4. **复杂度**：所有优化均为 O(n²) 或更低级别（n = 四元式数量）。对于教学级编译器，规模足够小到可以接受。
